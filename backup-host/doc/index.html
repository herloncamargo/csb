Bem-vindo a documentação do Servidor Central de Backup!¶

O Servidor Central de Backup é constituído de dois scritps principais: “backup-server.py” e “backup-host.py”. O script “backup_server.py” é instalado na máquina que será o servidor de backup, enquanto que o script “backup_host.py” é instalado nos hosts que serão alvo de backup.

O script “backup_host.py” é um daemon que fica rodando indefinidamente a espera de contato do servidor de backup, sendo portanto executado em segundo plano. Sua função consiste em receber uma solicitação do host servidor de backups contendo uma relação de arquivos que deverão ser exportados para o servidor, onde serão armazenados como backup.
